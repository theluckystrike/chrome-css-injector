# Contributing to chrome-css-injector

Thank you for your interest in contributing! This document outlines the process for contributing to this project.

## Reporting Bugs

**Before reporting a bug:**
- Search existing issues to avoid duplicates
- Test with the latest version to see if the issue is already fixed

**To report a bug:**
1. Use the [Bug Report](https://github.com/theluckystrike/chrome-css-injector/issues/new?template=bug_report.md) template
2. Describe the issue clearly with steps to reproduce
3. Include browser version and extension version
4. Add screenshots if applicable

## Submitting Pull Requests

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/my-feature`
3. Make your changes following the code style
4. Add tests if applicable
5. Ensure all tests pass
6. Commit with clear messages: `git commit -m "feat: add injection feature"`
7. Push to your fork and submit a PR

## Code Style

- Use TypeScript for new code
- Follow existing code formatting
- Add JSDoc comments for public APIs
- Keep functions small and focused
